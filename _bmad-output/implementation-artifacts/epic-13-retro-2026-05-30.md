## Insights clés

1. **Thin clients = contrat HTTP stable** — investir dans `/control/summary` et OpenAPI-like docs avant d’enrichir TUI/dashboard.
2. **RunEvent schema v1 extensible** — `HEARTBEAT` ajouté sans breaking change ; enum + factory + roundtrip test = checklist pour futurs types (`POSITION_UPDATE`, `KILL_REQUESTED`).
3. **TUI Java-first validé** — JLine3 + Jackson ; pas besoin de Rust pour l’atelier MVP.
4. **Laravel adjacent au reactor Maven** — `dashboard/vendor/` gitignored ; `composer install` requis localement — documenter dans onboarding.
5. **Stale detection = config ops** — default 120s inadapté H1+ ; runbook doit rappeler `stale-thresholds.json`.
