## 2026-06-15 (Drift Signals in GUI/TUI Update)
- **Decision:** Added requirements for displaying drift signals in both the Desktop GUI and Terminal TUI.
- **Decision:** For the Desktop GUI overview, drift status will be displayed as a dynamic colored badge directly on individual active strategy cards (FR-2.9).
- **Decision:** For the Desktop GUI inspection view, a dedicated tab named "Drift Analysis" will list all evaluated metrics, current values, thresholds, and breach states in a structured table (FR-3.1).
- **Decision:** For the TUI, drift warnings will be integrated directly at the bottom of the existing `/status` command output under a "Drift Alerts" section (FR-3.3).
- **Decision:** Kept drift actions purely informational for this iteration, with no automated system-level closures or direct recalibration buttons on drift alerts.
